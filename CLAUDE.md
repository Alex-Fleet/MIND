# MIND: MIND Is Not Diary — 用户偏好（永久兜底）

> 此文件由MIND维护。hook 挂了也能被 Claude Code 读取。

## 🔴 铁律（每次张嘴前先读）
- **问什么答什么，说什么做什么。不问不答，不说不错。**
- **禁止加戏：用户没问的不要分析、不要建议、不要补充。**

## 六条铁律
1. **Git 由我决定**：可以改代码，但 commit、版本号、分支操作——我说了算。没指令不许 commit。
2. **Plan 先于 Code**：先 plan，批准后动手
3. **架构干净**：不堆"屎山"，代码可维护
4. **ARCHITECTURE.md 同步更新**：结构改动必须写进去
5. **README 写 changelog**：每个版本记录清楚
6. **不请示不做**：任何没明确说的事，不能做。可以提意见、可以随便想——但执行前必须请示。

## 工作风格
- Git由用户决定，不擅自commit
- Plan先于Code，批准后动手
- 架构干净，不堆屎山
- ARCHITECTURE.md同步更新
- README写changelog

## 干活流程
1. **写/检查 Todo** — 把任务拆成清单，标优先级
2. **起子代理** — 复杂任务派给子代理做
3. **Review** — 检查子代理输出
4. **干活** — 主代理执行修改
5. **子代理测试** — 改完让子代理跑测试验证

## 沟通
- 中文交流，直接不废话
- 用具体例子指出问题
- 方案要彻底，不要图方便

## 代码风格
- 修根不修叶——不在产出物上手修，分析根因改代码/prompt/流程
- 发现漏洞不补一个，改架构让漏洞长不出来
- 表驱动替代分支嵌套

## Prompt工程哲学
- 定性>定量
- 用读者体验语言描述质量要求
- 【】分隔符组织规则
- 给出具体反例
